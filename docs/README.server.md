# HomeGate — домашний сервер Димы

> Читай это первым делом при подключении. Здесь всё, что нужно знать о машине.

## Что это и чьё это

Домашний сервер **у Димы дома** (друг Тигры). Тигра — админ этого стека,
Дима — хозяин дома и пользователь.

**Это НЕ инфраструктура SWIPE.** Никаких пересечений с monitoring.swipeandev.ru,
skynet, prilutsky.ru: отдельные токены, отдельная память, отдельные бэкапы.
Не тащи сюда ключи из рабочего контура и не отправляй отсюда данные туда.

Назначение: управление домашней автоматикой, мониторинг датчиков, MCP-гейт,
память хозяина дома, веб-дашборды.

## Что развёрнуто

Раскладка гибридная: инфраструктура нативно, сервисы в Docker.

| Сервис | Где | Порт | Автозапуск |
|---|---|---|---|
| nginx | нативно | 80, 443 | systemd |
| node_exporter | нативно | 9100 | systemd |
| homegate (MCP) | нативно, venv | 127.0.0.1:8800 | systemd |
| Qdrant | Docker | 127.0.0.1:6333 | compose |
| Prometheus | Docker | 127.0.0.1:9090 | compose |
| Grafana | Docker | 127.0.0.1:3000 | compose |

Наружу торчат только 80/443 и SSH. Всё остальное — на localhost, доступ через nginx.
**Qdrant намеренно не проксируется наружу.**

## Архитектура гейта — почему так, а не клон

Клон рабочего `/opt/mcpserver` с голым `run_command` для дома **сознательно
отвергнут**. Причина: дома цена ошибки выше, чем на рабочем сервере — там упадёт
сайт, здесь останется без отопления живой человек, и чинить некому.

Принципы:

1. **Инструменты типизированные, а не shell.** При `run_command` модель сама
   решает, какой командой добиться цели, и может ошибиться как угодно.
   При `device_control(entity, action)` максимум — не та лампа.

2. **Чтение свободно, запись — по белому списку.** Разобраться в проблеме можно
   без спроса. Менять состояние дома — только для сущностей из
   `homeassistant.write_whitelist`.

3. **Домены `lock`, `climate`, `water_heater`, `valve`, `alarm_control_panel`
   запрещены жёстко** — в коде, а не в конфиге. Даже если кто-то впишет их
   в whitelist, `device_control` откажет. Замки, отопление, вода, щит —
   только руками хозяина.

4. **Роли разделены токенами.** У Димы нет shell физически: `tools/list`
   не отдаёт `run_command`, а прямой вызов возвращает отказ.

5. **Аудит.** Каждый вызов shell и каждое управление устройством пишется
   в `audit.log` с именем вызывающего.

### Инструменты

Дима (`role: user`) — 7 штук:
`memory_save`, `memory_search`, `memory_list`, `home_state`, `sensor_history`,
`home_anomalies`, `device_control`

Тигра (`role: admin`) — те же плюс `run_command`, `service_status`.

`home_anomalies` — не «покажи датчики», а «что выглядит не так»: молчащие
больше суток сенсоры, батареи ниже 20%, недоступные устройства. То, что человек
не заметит, а модель заметит при первом вопросе.

## Учётные данные

Все токены выпущены заново на этой машине, ничего не переиспользовано.
Лежат в `/opt/homegate/config/config.json` и `/opt/monitoring/.env`.

- `dima_user` — роль user
- `admin_tigra` — роль admin
- `qdrant_api` — ключ Qdrant (проверено: без него 401)
- Grafana admin — пароль в `/opt/monitoring/.env`
- `ha_bridge` — выпущен заранее под Home Assistant, ещё не задействован

## Частые операции

```bash
# статус всего
systemctl status homegate nginx node_exporter
docker ps

# перезапуск гейта после правки кода или конфига
systemctl restart homegate
tail -f /opt/homegate/logs/service.log

# кто что делал
tail -50 /opt/homegate/logs/audit.log

# контейнеры
cd /opt/monitoring && docker compose restart
cd /opt/qdrant && docker compose logs --tail=50

# проверка гейта
curl -s http://127.0.0.1:8800/claude-mcp/health

# цели прометея
curl -s "http://127.0.0.1:9090/api/v1/targets?state=active" | python3.11 -m json.tool
```

## Грабли этой машины

- **SELinux Enforcing.** Новые каталоги под логи требуют контекста, иначе
  systemd молча падает с `status=209/STDOUT`:
  `semanage fcontext -a -t var_log_t "/path(/.*)?" && restorecon -Rv /path`
  Для бинд-маунтов в Docker — суффикс `:Z` в compose.
- **nginx 1.20**, не 1.25: `listen 443 ssl http2;` одной строкой,
  отдельная директива `http2 on;` не поддерживается.
- **Обёртка Team MCP рвёт SSH на 25 секундах.** Процесс на сервере при этом
  продолжает работать. Долгие команды — только через `nohup ... > /tmp/x.log 2>&1 &`,
  прогресс смотреть отдельным вызовом. Не принимай медленный dnf за зависший
  и не убивай его (я один раз так сделал зря).
- **Celeron N5105 медленный.** Установка пакетов и скачивание образов идут
  минутами. Это нормально, не признак поломки.
- Дефолтный `server`-блок вырезан из `nginx.conf`, бэкап рядом (`.bak`).

(Zigbee / Wi-Fi / Modbus / провод). От этого зависит:
нужен ли USB-стик и Zigbee2MQTT, нужен ли MQTT-брокер, что класть в compose.

API вслепую.
